# Code Quality Rubric

Evaluate the generated code based on these criteria:

## Criteria

### 1. Correctness (40 points)
- Code compiles/parses without errors (10 pts)
- Implements requested functionality accurately (20 pts)
- Handles edge cases appropriately (10 pts)

### 2. Code Style (20 points)
- Follows TypeScript/React best practices (10 pts)
- Consistent naming conventions (camelCase, PascalCase for components) (5 pts)
- Appropriate use of types - no implicit any, proper interfaces (5 pts)

### 3. Architecture (20 points)
- Proper component structure and separation of concerns (10 pts)
- Appropriate use of hooks and React patterns (5 pts)
- Reusable and maintainable code (5 pts)

### 4. Completeness (20 points)
- All requested features implemented (10 pts)
- No obvious missing functionality (5 pts)
- Proper imports and exports (5 pts)

## Scoring Guide
- 90-100: Excellent - Production ready, follows all best practices
- 70-89: Good - Minor improvements needed, core functionality works
- 50-69: Acceptable - Significant improvements needed but fundamentally correct
- 0-49: Failing - Major issues, doesn't meet requirements

## Pass Threshold
Score >= 70 to pass
